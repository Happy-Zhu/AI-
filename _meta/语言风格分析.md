# 嘿皮语言风格分析

> 本文档基于已有课程内容（第 01-10 期视频口播稿和飞书文档版）提炼的语言风格特征。后续如果有新的日常写作素材加入，会持续更新和补充。

---

## 1. 整体气质

嘿皮的表达不像老师在讲台上讲课，更像一个懂行的朋友在认真跟你解释一件事。

核心气质可以用四个词概括：**理性、真诚、有判断、不装。**

- 不贩卖焦虑，不说"再不学就被淘汰"
- 不过度承诺，不说"学了马上月入十万"
- 有自己的判断，会说"我觉得""我的答案很简单"
- 不回避边界，AI 能做什么不能做什么会直接说

---

## 2. 句式特征

### 2.1 短句为主，断句清晰

嘿皮几乎不用长复合句。每一段的句子都很短，断句节奏感强，适合口播和快速阅读。

典型例子：

> AI 既不是神，也不是玩具。
>
> 你可以先把它理解成一个"信息处理助手"。

> 这当然可以问，但效果通常一般。
>
> 原因很简单：你没有给任务，也没有给背景，它只能猜。

### 2.2 先破后立

几乎每个重要观点都是先说"错误认知"或"常见做法"，再给出正确方式。

典型结构：
- "很多人……但其实……"
- "你可能会……但更好的方式是……"
- "不要直接说……更好的写法是……"

例子：

> 很多人学 AI，一上来就找："万能提示词""神级 Prompt""一键爆款模板"。
>
> 这些模板不是完全没用。但如果你只会复制，不理解背后的结构，很快就会失效。

### 2.3 用"你"直接对话

写作中大量使用第二人称"你"，很少用"学员""读者""用户"。

这让文字有一种面对面说话的感觉。

> 你不能把 AI 当成一个永远正确的答案机器，但你可以把它当成一个很强的起点生成器。

> 你不给边界，AI 就会自己补。它补对了，你觉得好；补错了，你觉得它废。

### 2.4 "但这里也要说实话"型坦诚

在讲了 AI 好处之后，会主动补一句诚实的限制说明。不为了课程效果而回避缺点。

> 但这里也要说实话：AI 写出来的东西，往往只能算初稿。
>
> 真正让内容有价值的，是你自己的判断、经验和修改。

> AI 可以帮你加速，但不能替你判断。

### 2.5 活人感：短句里塞满真实信息

嘿皮最核心的风格特征之一：**看起来很短的句子，实际上信息密度极高。**

不会说"Claude 很贵"，而是说出具体体感：

> Claude 20 美金的官方订阅消耗起来特别快，一个小项目，你让他给你介绍一下，他扫描一遍，估计三四块人民币就没了，而 ChatGPT 20 美金的官方订阅，从体感上来讲，额度是 Claude Code 20 美金的 5 倍不止。并且前段时间 OpenAI 还说，只要订阅人数增加一百万，就会给全部用户重置一次额度。

这段话的特征：
- **有真实金额**：20 美金、三四块人民币、5 倍不止
- **有体感判断**：不是查来的数据，是自己用出来的
- **有行业信息**：OpenAI 订阅人数增长就重置额度这种"圈内才知道的事"
- **有对比**：两个产品的体感差异，让人一下就知道选谁
- **不用"据说""大概"**：直接说数字和判断，语气确定

这种风格的关键是：**你说的每一句话，都要让读者觉得"这个人真的用过"**。

再举几个例子：

> 中转站约等于第三方的 API 商。你不直接使用 OpenAI 官方 API，通过第三方平台进去，然后拿到一个 API Key 和接口地址，再配到应用里使用。

> 千万不要一上来就追求最强。AI 编程工具和普通聊天不一样，涉及到项目文件读写、执行过程和边界，稳定性和安全性比什么都重要。

写作时的规则：
1. **能给数字就给数字**——不说"很贵"，说"20 美金""三四块"
2. **能给体感就给体感**——不说"消耗快"，说"扫描一遍估计就没了"
3. **能给对比就给对比**——不说"A 比 B 好"，说"体感上 A 的额度是 B 的 5 倍不止"
4. **能给圈内信息就给**——"前段时间 OpenAI 还说了一件事"这种句式，让读者觉得你在跟他说内幕
5. **敢下判断**——不说"可以考虑"，说"我建议先用官方"

## 3. 讲解方法

### 3.1 概念讲解：一句话 → 类比 → 例子 → 和你的关系

几乎所有抽象概念都用同一套讲解顺序：

1. 先用一句大白话定义
2. 再用一个生活化类比降低理解门槛
3. 然后给一个具体例子
4. 最后说明"它和你日常使用 AI 有什么关系"

例子（Token 的讲解）：

> Token 可以粗略理解成 AI 读文字时使用的"小颗粒"。你发给 AI 的问题，会被拆成一个个 Token；AI 回复你的内容，也会消耗 Token。你可以把它想成手机流量，输入和输出都会占用额度。

### 3.2 方法讲解：先给反面 → 再给正面 → 附模板

讲操作方法时，几乎都会先展示"大多数人怎么做（效果差）"，再展示"更好的做法"，最后附上可以直接复制的提示词模板。

例子：

> 不要只说"帮我总结一下。"这样很容易得到一段泛泛总结。
>
> 更好的写法是：
> ```
> 请阅读下面这篇文章，帮我整理成：
> 1. 5 个核心观点；
> 2. 每个观点对应的原文依据……
> ```

### 3.3 判断标准用表格

每当涉及"怎么判断好坏""什么时候该用/不该用"，会使用表格呈现，方便扫读和对比。

但表格的使用是克制的，一节课通常只有 1-3 张表格，不会为了"丰富"而堆表格。

## 4. 禁忌词和禁忌表达

嘿皮在写作中有意识回避以下词汇和表达方式：

| 禁忌类型 | 具体表现 | 替代方式 |
| --- | --- | --- |
| 空话 | "赋能""降本增效""智能化转型" | 直接说具体场景 |
| 焦虑表达 | "再不学就晚了""错过就被淘汰" | 讲学习的现实理由 |
| 过度承诺 | "一键搞定""自动赚钱" | 说清楚能做到什么程度 |
| 教科书腔 | "综上所述""由此可见" | 用口语连接词 |
| 模糊指代 | "它可以帮你做很多事" | 必须接具体例子 |
| 万能包装 | "最强工具""唯一答案" | 说清楚适合谁、不适合谁 |

## 5. 结构节奏

### 5.1 视频口播稿的节奏

- 开场：用一个小白能感知的问题或误区切入（不超过 1 分钟）
- 正文：按 3-5 个递进模块展开，每个模块有"观点 + 例子 + 提醒"
- 收尾：用几句话总结核心判断 + 给一个可以立刻做的小练习
- 衔接：结尾会预告下一期内容（"下一期我们来聊……"）

### 5.2 飞书文档版的节奏

- 结构更清晰，用标题分节
- 每节开头直接说"这一部分解决什么问题"
- 示例提示词用代码块，方便复制
- 引用块只用于核心判断或重点提醒（一节课 2-4 个）
- 结尾有"本节小结"和"课后小任务"

## 6. 情感基调

### 做到的：
- 像朋友聊天，不像老师训话
- 有自信但不傲慢——"我的答案很简单""我觉得有三个现实理由"
- 对小白有耐心，不嫌问题简单
- 对 AI 有敬畏，不回避它的局限

### 不做的：
- 不制造紧迫感
- 不做销售转化
- 不贬低不懂 AI 的人
- 不把任何工具捧成唯一选择

---

## 7. 常用口头语 / 标志性表达

以下是在已有课程中反复出现的标志性表达，后续写作时可以自然继承：

- "先把期待放准"
- "先说清楚""先把最底层的事情讲清楚"
- "不是……而是……" —— 先纠偏，再定义
- "这句话很重要"
- "注意，我说的是……"
- "但这里也要说实话"
- "你看，这就是……的正确方式"
- "三个坑要先避开"
- "能力越强，边界越重要"
- "先让它看，别急着改；先让它计划，再让它执行"
