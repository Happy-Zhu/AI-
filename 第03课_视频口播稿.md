# 第 03 课视频口播稿：AI 常见名词扫盲：模型、Token、上下文、Prompt、API

## 基本信息

- **建议时长**：8-10 分钟
- **主讲开场**：大家好，我是嘿皮
- **课程基调**：理性、真诚、有干货
- **本节目标**：让小白听懂 AI 入门里最常见的几个词，后面看工具、教程和文章时不再一脸懵

## 口播正文

### 1. 开场：先别被名词吓住

大家好，我是嘿皮。

前两节课，我们先讲了 AI 和普通人的关系，也讲了 AI 为什么不是搜索引擎，而更像一个根据上下文生成回答的“推理机”。

这一节课，我们来做一次名词扫盲。

你只要开始接触 AI，很快就会遇到一堆词：

模型、Token、上下文、Prompt、API、知识库、Agent。

这些词看起来很技术，但你不需要一开始就学成工程师。

你只需要先知道：它们分别在 AI 使用里扮演什么角色，和你有什么关系。

这一节课，我们先抓最重要的五个：模型、Token、上下文、Prompt、API。

### 2. 模型：AI 背后的“大脑”

第一个词，模型。

你可以先把模型理解成 AI 背后的“大脑”。

比如你打开 ChatGPT、DeepSeek、Kimi、豆包，其实你不是在和一个网页聊天，而是在和背后的大模型交互。

不同模型，就像不同能力的人。

有的更擅长写作，有的更擅长推理，有的更擅长代码，有的中文体验更好，有的长文档处理能力更强。

所以模型不是越贵越好，也不是越新越适合你。

对小白来说，先记住一句话：

模型决定了 AI 的基础能力上限，但你的提问方式会影响它能发挥多少。

就像你请了一个很厉害的老师，但你问题问得很乱，它也很难直接给你最想要的答案。

### 3. Token：AI 读写文字的小颗粒

第二个词，Token。

上一节课我们已经简单提过，这里再把它讲清楚一点。

Token 可以粗略理解成 AI 读写文字时使用的小颗粒。

你发给 AI 的内容会被拆成 Token，AI 回复你的内容也会消耗 Token。

一个很直观的类比是：Token 有点像手机流量。

你输入越多，输出越长，消耗就越多。

为什么小白要知道 Token？

因为它和三件事有关。

第一，和费用有关。很多 API 或高级工具，会按输入输出的 Token 计费。

第二，和上下文有关。AI 一次能记住和处理的信息不是无限的。

第三，和提问方式有关。你给的信息太少，它理解不够；你给的信息太乱，它也会浪费空间。

所以不要一上来把几十页内容全丢进去，也不要只丢一句“帮我看看”。

更好的方式是：先说明目标，再给关键资料，再告诉它你想要什么输出。

### 4. 上下文：AI 当前能看到的信息

第三个词，上下文。

上下文就是 AI 当前能看到、能参考的信息。

包括你刚刚问的问题、你提供的资料、前面的对话记录，以及你给它的限制条件。

你可以把上下文想成一张工作台。

AI 要帮你干活，得先看见工作台上有什么。

如果你什么都不给，只说“帮我写一篇文章”，它只能猜。

如果你告诉它：

我要写给完全小白看；

主题是普通人为什么要学 AI；

风格要理性真诚，不要贩卖焦虑；

字数控制在 1500 字；

请先给我提纲。

这时候它的回答就会稳定很多。

所以小白使用 AI 的时候，要学会给上下文。

不是话越多越好，而是关键信息越清楚越好。

### 5. Prompt：你给 AI 的任务说明

第四个词，Prompt。

Prompt 常被翻译成提示词，但我觉得对小白来说，可以先把它理解成“任务说明”。

你每次发给 AI 的话，其实都是 Prompt。

一句“帮我写一篇文章”是 Prompt。

一段很完整的任务说明也是 Prompt。

两者的区别在于，前者太模糊，后者更清楚。

一个基础好用的 Prompt，通常包含五个部分：

角色、任务、背景、要求、格式。

比如：

你是一位擅长给小白讲课的 AI 老师。

请帮我解释什么是 Token。

听众是完全没有技术基础的新手。

要求语言口语化，举一个生活例子。

输出格式为三段：一句话解释、生活类比、使用注意事项。

这就比单纯问“什么是 Token”要稳定得多。

### 6. API：让软件调用 AI 的接口

第五个词，API。

API 这个词对小白最容易显得吓人。

其实你可以先把 API 理解成一个“接口”。

网页版 ChatGPT 是你手动打开网页，自己输入问题。

API 则是让软件、网站、工作流自动去调用 AI。

比如一个写作工具里有 AI 改写功能；

一个客服系统能自动回复用户；

一个工作流能自动抓取热点、总结内容、生成配图；

这些背后都可能用到 API。

所以 API 的意义是：让 AI 不只停留在聊天窗口里，而是进入真实工具和流程里。

但 API 也有风险。

比如 API Key 不能随便发给别人，因为它相当于调用权限；调用 API 可能产生成本；把隐私数据传给任何工具前，都要先想清楚边界。

小白不需要马上会写 API，但要知道它为什么重要。

### 7. 五个词串起来怎么理解？

现在我们把这五个词串起来。

你给 AI 发了一段 Prompt，也就是任务说明。

这些文字会被拆成 Token。

AI 会结合当前上下文，也就是它能看到的信息，交给模型去生成回答。

如果你不是手动在网页里操作，而是让某个软件自动调用 AI，那背后可能就会用到 API。

所以它们不是五个孤立概念，而是一条完整链路。

Prompt 是你怎么说；

Token 是 AI 怎么读；

上下文是 AI 当前知道什么；

模型是 AI 用什么能力生成；

API 是软件怎么调用 AI。

### 8. 结尾：小白怎么练？

这一节课的练习很简单。

你打开 ChatGPT，直接问它：

```text
请用小白能听懂的话，分别解释模型、Token、上下文、Prompt、API。
每个词都要包含：一句话解释、生活类比、和普通用户的关系。
```

然后你再追问：

```text
请把这五个词串成一个完整流程，解释我向 AI 提问时，背后大概发生了什么。
```

你不用一下子记住所有技术细节。

先把这些词和你的使用场景连起来。

下一次你再看到 AI 工具介绍里写“支持长上下文”“按 Token 计费”“开放 API”“优化 Prompt”，你就不会完全陌生了。
