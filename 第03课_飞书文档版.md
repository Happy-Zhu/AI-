# 第 03 课：AI 常见名词扫盲：模型、Token、上下文、Prompt、API

## 1. 先别被名词吓住

只要开始接触 AI，你很快就会遇到一堆词：

模型、Token、上下文、Prompt、API、知识库、Agent。

这些词看起来很技术，但你不需要一开始就学成工程师。

对完全小白来说，第一步不是把每个词研究到特别深，而是先知道：

**它们分别在 AI 使用里扮演什么角色，和我有什么关系。**

这一课先讲最常见、最基础的五个词：

模型、Token、上下文、Prompt、API。

## 2. 模型：AI 背后的“大脑”

你可以先把模型理解成 AI 背后的“大脑”。

比如你打开 ChatGPT、DeepSeek、Kimi、豆包，其实你不是在和一个网页聊天，而是在和背后的大模型交互。

不同模型，就像不同能力的人。

有的更擅长写作，有的更擅长推理，有的更擅长代码，有的中文体验更好，有的长文档处理能力更强。

所以模型不是越贵越好，也不是越新越适合你。

> 模型决定了 AI 的基础能力上限，但你的提问方式会影响它能发挥多少。

就像你请了一个很厉害的老师，但你问题问得很乱，它也很难直接给你最想要的答案。

## 3. Token：AI 读写文字的小颗粒

Token 可以粗略理解成 AI 读写文字时使用的小颗粒。

你发给 AI 的内容会被拆成 Token，AI 回复你的内容也会消耗 Token。

一个直观类比是：Token 有点像手机流量。

你输入越多，输出越长，消耗就越多。

小白知道 Token，主要是为了理解三件事：

| 关系 | 小白版解释 |
| --- | --- |
| 费用 | 很多 API 或高级工具会按输入输出的 Token 计费 |
| 记忆长度 | AI 一次能记住和处理的信息不是无限的 |
| 提问方式 | 信息太少不够用，信息太乱会浪费上下文 |

所以，不要一上来把几十页内容全丢进去，也不要只丢一句“帮我看看”。

更好的方式是：先说明目标，再给关键资料，再告诉它你想要什么输出。

## 4. 上下文：AI 当前能看到的信息

上下文就是 AI 当前能看到、能参考的信息。

它包括：

- 你刚刚问的问题；
- 你提供的资料；
- 前面的对话记录；
- 你给它的限制条件。

你可以把上下文想成一张工作台。

AI 要帮你干活，得先看见工作台上有什么。

如果你什么都不给，只说：

```text
帮我写一篇文章。
```

它只能猜。

如果你这样说：

```text
我要写给完全小白看。
主题是普通人为什么要学 AI。
风格要理性真诚，不要贩卖焦虑。
字数控制在 1500 字。
请先给我提纲。
```

它的回答就会稳定很多。

所以小白使用 AI 时，要学会给上下文。

不是话越多越好，而是关键信息越清楚越好。

## 5. Prompt：你给 AI 的任务说明

Prompt 常被翻译成提示词。

但对小白来说，可以先把它理解成：**你给 AI 的任务说明**。

你每次发给 AI 的话，其实都是 Prompt。

一句“帮我写一篇文章”是 Prompt。

一段完整任务说明也是 Prompt。

区别在于，前者太模糊，后者更清楚。

一个基础好用的 Prompt，通常包含五个部分：

```text
角色 + 任务 + 背景 + 要求 + 格式
```

比如：

```text
你是一位擅长给小白讲课的 AI 老师。
请帮我解释什么是 Token。
听众是完全没有技术基础的新手。
要求语言口语化，举一个生活例子。
输出格式为三段：一句话解释、生活类比、使用注意事项。
```

这就比单纯问“什么是 Token”要稳定得多。

## 6. API：让软件调用 AI 的接口

API 这个词对小白最容易显得吓人。

你可以先把 API 理解成一个“接口”。

网页版 ChatGPT 是你手动打开网页，自己输入问题。

API 则是让软件、网站、工作流自动去调用 AI。

比如：

- 一个写作工具里有 AI 改写功能；
- 一个客服系统能自动回复用户；
- 一个工作流能自动抓取热点、总结内容、生成配图；
- 一个公司内部系统能自动读取资料并生成报告。

这些背后都可能用到 API。

所以 API 的意义是：让 AI 不只停留在聊天窗口里，而是进入真实工具和流程里。

但 API 也有风险。

API Key 不能随便发给别人，因为它相当于调用权限；调用 API 可能产生成本；把隐私数据传给任何工具前，都要先想清楚边界。

小白不需要马上会写 API，但要知道它为什么重要。

## 7. 五个词串起来怎么理解？

现在把这五个词串起来。

你给 AI 发了一段 **Prompt**，也就是任务说明。

这些文字会被拆成 **Token**。

AI 会结合当前 **上下文**，也就是它能看到的信息，交给 **模型** 去生成回答。

如果你不是手动在网页里操作，而是让某个软件自动调用 AI，那背后可能就会用到 **API**。

换成一句更直白的话：

> Prompt 是你怎么说；Token 是 AI 怎么读；上下文是 AI 当前知道什么；模型是 AI 用什么能力生成；API 是软件怎么调用 AI。

## 8. 课后小任务

打开 ChatGPT，或者你常用的 AI 工具，问它：

```text
请用小白能听懂的话，分别解释模型、Token、上下文、Prompt、API。
每个词都要包含：一句话解释、生活类比、和普通用户的关系。
```

然后继续追问：

```text
请把这五个词串成一个完整流程，解释我向 AI 提问时，背后大概发生了什么。
```

你不用一下子记住所有技术细节。

先把这些词和你的使用场景连起来。

下一次你再看到 AI 工具介绍里写“支持长上下文”“按 Token 计费”“开放 API”“优化 Prompt”，你就不会完全陌生了。

## 9. 本节小结

这一课主要帮你建立一张名词地图。

模型，是 AI 背后的能力基础。

Token，是 AI 读写文字的小颗粒。

上下文，是 AI 当前能看到的信息。

Prompt，是你给 AI 的任务说明。

API，是让软件调用 AI 的接口。

这些词并不只是技术名词，它们会直接影响你怎么提问、怎么选工具、怎么判断成本和风险。
