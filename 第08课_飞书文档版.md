# 第 08 课：ChatGPT 到 Codex：让 AI 从“回答问题”到“推进任务”

## 1. Codex 不是“更会聊天的 ChatGPT”

前面几节课，我们一直在讲 ChatGPT。

ChatGPT 很适合帮你解释概念、整理资料、写提纲、改表达。

但 Codex 不是简单的“另一个聊天窗口”。

你可以先把 Codex 理解成一种 **AI 编程 Agent**。

它更重要的变化是：它可以进入你的项目目录里工作。

它能读取文件、理解目录结构、修改内容、运行命令，并根据报错继续调整。

> ChatGPT 更适合陪你想，Codex 更适合帮你把任务往前推。

哪怕你不是程序员，也值得理解它。

因为它代表了 AI 从“回答问题”进入“执行任务”。

## 2. ChatGPT 和 Codex 的核心区别

| 对比项 | ChatGPT | Codex |
| --- | --- | --- |
| 更像什么 | 陪你讨论的老师 | 能进项目里动手的助理 |
| 工作位置 | 对话窗口 | 本地/云端项目目录 |
| 主要能力 | 解释、写作、总结、规划 | 读文件、改文件、跑命令、修问题 |
| 输入方式 | 你粘贴内容或上传文件 | 直接读取工作区文件 |
| 风险点 | 答案可能不准 | 可能改错文件、误执行命令 |
| 正确心态 | 多追问、多核验 | 给边界、看计划、验结果 |

这张表的重点是：Codex 的能力更靠近“执行”。

能力越强，边界越重要。

## 3. 普通人能用 Codex 做什么？

不要一听 Codex 就觉得它只属于程序员。

普通人也可以用它做很多明确的小任务。

| 场景 | 可以让 Codex 做什么 | 难度 |
| --- | --- | --- |
| 自媒体 | 做一个选题管理网页 | 低 |
| 课程制作 | 整理 Markdown 文件，生成课程目录 | 低 |
| 文件整理 | 批量重命名、格式转换、生成索引 | 中 |
| 个人工具 | 报价单生成器、客户信息表、小网页 | 中 |
| 代码项目 | 修 Bug、改样式、补文档、跑测试 | 高 |

这里最重要的一句话是：

**AI 编程的红利，不只是程序员的红利，而是普通人把想法变成工具的门槛降低了。**

## 4. Codex 使用的安全顺序

新手不要一上来就说：

```text
帮我优化这个项目。
```

这个指令太危险。

因为你没有说优化哪里，也没有限制它能不能改文件。

更稳的顺序是：

```text
先理解项目 -> 提出计划 -> 确认后执行 -> 解释改动 -> 给出检查方法
```

### 第一步：只读分析，不修改

```text
请先阅读当前目录，理解项目结构。
不要修改任何文件。
请输出：
1. 项目主要文件；
2. 每个文件的作用；
3. 当前项目可能存在的问题；
4. 你建议下一步优先处理的 3 件事。
```

### 第二步：让它提出计划

```text
请基于上面的分析，给出修改计划。
要求：
1. 说明要修改哪些文件；
2. 每个文件为什么要改；
3. 风险是什么；
4. 暂时不要执行。
```

### 第三步：确认后再执行

```text
同意执行第 1 项和第 2 项。
只允许修改 README.md 和 index.html。
不要修改其他文件。
修改完成后，请说明改动内容和检查方法。
```

这才是新手比较安全的 Codex 用法。

## 5. 完整案例：让 Codex 整理课程文件夹

假设你有一个课程文件夹，里面有：

```text
第01课_视频口播稿.md
第01课_飞书文档版.md
第02课_视频口播稿.md
第02课_飞书文档版.md
AI扫盲小白课程_汇总索引.md
assets/
```

你想让 Codex 帮你整理索引。

不要直接说：

```text
帮我整理一下。
```

更好的提示词是：

```text
请阅读当前目录，帮我检查课程文件是否完整。
不要修改任何文件。
请输出：
1. 目前有哪些课；
2. 每课是否同时有视频口播稿和飞书文档版；
3. assets 里有哪些素材；
4. 汇总索引里缺少哪些文件路径；
5. 下一步建议怎么补。
```

如果它分析正确，再让它修改：

```text
请只修改 AI扫盲小白课程_汇总索引.md。
目标是补齐第 1-10 课的文件路径和素材状态。
不要修改课程正文。
修改前先列计划，等我确认。
```

这就是 Codex 的价值：它能跨文件处理任务。

## 6. 什么时候不要用 Codex？

Codex 很有用，但不是所有任务都适合。

| 不建议直接用的场景 | 原因 | 更稳做法 |
| --- | --- | --- |
| 重要项目大改 | 改错成本高 | 先备份，先小范围改 |
| 数据库、支付、权限 | 风险高 | 人工确认方案，再执行 |
| 隐私代码或客户资料 | 数据安全问题 | 不上传或脱敏处理 |
| 需求还没想清楚 | 它会按自己理解乱做 | 先用 ChatGPT 梳理需求 |
| 你完全看不懂结果 | 无法验收 | 先做低风险小任务 |

## 7. 小白练习路线

第一次接触 Codex，不要直接挑战大项目。

按这个顺序练：

1. 空文件夹里创建 README；
2. 生成一个简单 HTML 页面；
3. 修改页面样式；
4. 整理一组 Markdown 文件；
5. 做一个批量重命名脚本；
6. 让它分析一个小项目，不修改；
7. 让它在限定文件里做小改动。

练习标准不是“做多复杂”，而是你能不能做到：

- 给清楚目标；
- 给清楚边界；
- 看懂计划；
- 验收结果。

## 8. 本节小结

Codex 的重点不是“它会写代码”。

它的重点是：AI 可以进入文件和项目，帮你推进任务。

但越能执行，越需要边界。

新手记住一句话：

> 先让它看，别急着改；先让它计划，再让它执行。
