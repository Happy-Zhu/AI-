# 第 03 期视频口播稿：AI 世界里的 5 个关键词——搞懂了，后面学什么都不慌

## 基本信息

- **建议时长**：8-10 分钟
- **主讲开场**：大家好，我是嘿皮
- **表达基调**：理性、真诚、不堆名词、用一个完整故事串联五个概念
- **本期目标**：让小白听懂模型、Token、上下文、Prompt、API 这五个词，并且知道它们和自己有什么关系

## 口播正文

### 1. 开场：你是不是一看到英文词就头大？

大家好，我是嘿皮。

今天我们来做一件很多小白特别需要、但一直在逃避的事——**名词扫盲**。

你只要开始接触 AI，不管看教程、看文章、还是看工具介绍，很快就会碰到一堆英文词：

Model、Token、Context Window、Prompt、API、Agent、RAG……

看着就头大。

很多人到这一步就放弃了："算了，这些太技术了，我不适合学 AI。"

但说实话，这些词你不用学到工程师级别。你只需要花 10 分钟搞明白最核心的 5 个，后面学什么工具、看什么教程，都不会再一脸懵。

今天我们就讲这 5 个：**模型、Token、上下文、Prompt、API。**

而且我不会一个一个孤立地讲定义。我会用一个完整的故事串起来——就是你打开 ChatGPT 问了一个问题之后，背后到底发生了什么。

### 2. 你打开 ChatGPT 问了一个问题——背后发生了什么？

假设你打开 ChatGPT，输入了这么一句话：

"请用小白能听懂的话解释什么是 Token。"

然后按了回车。

你看到的是：AI 开始一个字一个字往外蹦，几秒钟就写出一段解释。

但在这几秒钟里，背后其实发生了一条完整的链路。我们用这个例子来逐个拆。

### 3. 第一个词：Prompt——你发出去的那句话

你输入的那句话——"请用小白能听懂的话解释什么是 Token"——这就是 **Prompt**。

Prompt 常被翻译成"提示词"，但我觉得这个翻译容易让人误解，好像是某种神秘咒语。

其实 Prompt 就是你给 AI 的任务说明。

每次你发给 AI 的话，都是 Prompt。

一句"帮我写篇文章"是 Prompt。一段很完整的任务描述也是 Prompt。

区别不在于长短，在于**你有没有把任务说清楚**。

你说"帮我写篇文章"——它只能猜你想要什么。

你说"我要写一篇给小白看的文章，主题是普通人为什么要学 AI，风格真诚不贩卖焦虑，控制在 1500 字，先给我提纲"——它就知道你要什么了。

Prompt 的质量，直接决定了 AI 回答的质量。后面第 06 期我们会专门讲怎么写好 Prompt，今天先记住这个概念。

### 4. 第二个词：Token——AI 怎么读你的话

你的 Prompt 发出去之后，AI 不会像人一样"读句子"。

它会先把你的文字拆成一个个小颗粒——这些小颗粒就叫 **Token**。

你可以把 Token 想成手机流量。

你发给 AI 的问题会消耗 Token，AI 回复你的答案也会消耗 Token。输入越多、输出越长，消耗就越大。

为什么小白要知道 Token？因为它和三件事直接相关：

第一，和钱有关。很多 API 和高级功能是按 Token 计费的。你用得越多，花得越多。

第二，和记忆有关。AI 一次能处理的 Token 是有上限的。聊天太长、塞的资料太多，它可能就记不住最前面的内容了。

第三，和你怎么提问有关。你给的信息太少，它理解不够；你给的信息太杂太乱，它也会浪费空间。

所以小白不需要算 Token，但要知道：**不要一口气塞几十页进去，也不要只给一句"帮我看看"**。

### 5. 第三个词：上下文——AI 当前能看到什么

你的文字被拆成 Token 之后，AI 就要开始工作了。

但它不是凭空生成答案的。它要参考一些信息——这些信息就是**上下文**。

上下文包括什么？包括你刚刚问的问题、你前面对话里说过的话、你上传的文件、你给它设定的限制条件。

你可以把上下文想成一张**工作台**。

AI 要帮你干活，得先看见工作台上有什么。工作台上东西越清楚，它干活越靠谱。

但这张工作台是有大小限制的。

如果对话太长、资料太多，最早的内容可能就被"挤出"工作台了——AI 看不到了，自然也就想不起来了。

这就是为什么你跟 AI 聊着聊着，它突然忘了你之前说的要求。不是它变傻了，而是工作台满了。

所以小白用 AI 的时候，要学会**管理上下文**：

- 一个任务开一个新对话，不要什么都塞在一起
- 重要的要求，放在对话的最前面或最后面，别埋在中间
- 对话太长时，可以把核心要求重新发一遍

### 6. 第四个词：模型——AI 背后的"大脑"

上下文准备好了，Token 拆完了，接下来就轮到**模型**上场了。

模型就是 AI 背后的"大脑"。

你打开 ChatGPT 的时候，其实是在和 OpenAI 的模型交互。你打开 DeepSeek，是在和 DeepSeek 的模型交互。Kimi 背后是 Kimi 的模型，Claude 背后是 Anthropic 的模型。

不同模型，就像不同能力的人。

有的写作强，有的推理强，有的代码强，有的中文体验好，有的擅长处理长文档。

所以模型不是越贵越好，也不是越新越适合你。

对小白来说，先记住一句话：

**模型决定了 AI 的能力上限，但你的 Prompt 会影响它能发挥多少。**

就像你请了一个很厉害的顾问，但你问题描述得很乱，顾问也很难直接给你想要的答案。

### 7. 第五个词：API——让 AI 进入真实工作

最后一个词，API。

这个词对小白来说最容易显得吓人。但你可以先把它理解成一个**"接口"**。

平时你用 ChatGPT，是自己打开网页、手动输入问题。

API 做的事情不同。它让**软件、网站和工作流自动去调用 AI**。

比如：
- 一个写作工具里有"AI 改写"按钮——背后调用了 API
- 一个客服系统能自动回复用户——背后调用了 API
- 一个工作流能自动抓热点、总结内容、生成配图——背后也是 API

所以 API 的意义是：**让 AI 不只停留在聊天窗口里，而是进入真实的工具和流程。**

小白不需要马上会写 API，但你要知道它为什么重要——因为以后你用的越来越多 AI 功能，背后可能都是通过 API 调用的。

另外有两个安全提醒：

第一，API Key 不能随便给别人。它相当于你的调用权限，别人拿了可以用你的额度、甚至访问你的数据。

第二，通过 API 传的数据，要注意隐私。客户资料、内部文件、个人敏感信息，传之前想想有没有风险。

### 8. 串起来：一个提问背后的完整链路

现在我们把这五个词串起来。

你给 AI 发了一段 **Prompt**（任务说明）。

这些文字被拆成 **Token**（AI 读取的小颗粒）。

AI 结合当前的**上下文**（它能看到的所有信息），交给**模型**（它的大脑）去推理和生成回答。

如果你不是手动在网页里操作，而是让某个软件自动调用 AI——那背后用的就是 **API**。

一句话总结：

> Prompt 是你怎么说，Token 是 AI 怎么读，上下文是 AI 当前知道什么，模型是 AI 用什么能力去推，API 是软件怎么调用 AI。

记住这条链路，以后你再看到 AI 工具介绍里写"支持 128K 上下文""按 Token 计费""开放 API""优化你的 Prompt"——你就全能看懂了。

### 9. 收尾

今天的课后任务很简单。

打开 ChatGPT 或者你常用的 AI 工具，直接问它：

```text
请用小白能听懂的话，分别解释模型、Token、上下文、Prompt、API。
每个词给我：一句话解释 + 一个生活类比 + 它和我日常使用 AI 有什么关系。
```

然后继续追问：

```text
请把这五个词串成一条链路，解释我向 AI 提问时，背后大概发生了什么。
```

你不用一下子全记住。先让这些词和你的使用经验连起来。
