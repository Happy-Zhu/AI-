# 第 03 期：AI 世界里的 5 个关键词——搞懂了，后面学什么都不慌

## 1. 你不是学不会 AI，你只是被名词吓住了

只要开始接触 AI，你很快就会在各种教程、文章和工具介绍里遇到一堆英文词：

Model、Token、Context Window、Prompt、API、Agent、RAG……

看着就头大。很多人到这一步就放弃了："这些也太技术了，我真的不适合学 AI。"

但说实话，这些词你不需要学到工程师级别。

**你只需要搞明白最核心的 5 个，后面学什么工具、看什么教程都不会再一脸懵。**

今天就讲这 5 个：模型、Token、上下文、Prompt、API。

但我不会一个一个孤立地念定义——那样太无聊了，你也记不住。

我换一个讲法：**用一个完整的故事串起来。**

故事很简单——你打开 ChatGPT，问了一个问题，按了回车。在你等它回答的那几秒钟里，背后到底发生了什么？

## 2. 故事开始：你向 ChatGPT 提了一个问题

假设你输入了这么一句话：

```text
请用小白能听懂的话解释什么是 Token。
```

按了回车。

你看到的是：AI 开始一个字一个字往外蹦，几秒钟就写出了一段解释。

但在这几秒钟里，背后发生了一条完整的链路。我们现在来一步步拆。

## 3. 第一步：Prompt——你发出去的那句话

你输入的那句话，就是 **Prompt**。

Prompt 常被翻译成"提示词"，但这个翻译容易让人误解，好像是某种神秘咒语。

其实 Prompt 就是**你给 AI 的任务说明**，没有任何玄学成分。

每次你发给 AI 的话，都是 Prompt。一句"帮我写篇文章"是 Prompt。一段很完整的任务描述也是 Prompt。

区别不在于长短，在于你有没有把任务说清楚。

我举个对比：

```text
❌ 帮我写篇文章。
```

```text
✅ 我要写一篇给 AI 小白看的文章。
   主题是"普通人为什么要学 AI"。
   风格要真诚、有干货，不贩卖焦虑。
   控制在 1500 字以内。
   请先给我提纲，不要直接写全文。
```

第一个，AI 只能猜你想要什么。第二个，AI 清楚地知道任务、读者、风格、字数和输出形式。

> Prompt 的质量，直接决定了 AI 回答的质量。

后面第 06 期会专门讲怎么写好 Prompt。今天先知道这个概念就够了。

## 4. 第二步：Token——AI 怎么"读"你的话

你的 Prompt 发出去之后，AI 不是像人一样"读句子"。

它会先把你的文字拆成一个个小颗粒。这些小颗粒就叫 **Token**。

一个直观的类比：**Token 像手机流量。**

你发给 AI 的问题会消耗 Token，AI 回复你的答案也会消耗 Token。输入越多、输出越长，消耗就越大。

小白为什么要知道 Token？因为它和三件事直接相关：

**① 和钱有关。** 很多 API 和高级功能是按 Token 计费的。你用得越多，花得越多。

**② 和记忆有关。** AI 一次能处理的 Token 是有上限的。聊天太长、塞的资料太多，它可能就记不住前面的内容了。

**③ 和提问方式有关。** 你给的信息太少，它理解不够；你给的信息太杂太乱，它也会浪费空间。

实际使用的建议：不要一口气塞几十页资料进去，也不要只给一句"帮我看看"。

**先说明目标，再给关键资料，再告诉它你想要什么输出。** 这样最省 Token，也最有效。

## 5. 第三步：上下文——AI 当前能看到什么

Token 拆完之后，AI 就要开始工作了。

但它不是凭空生成答案。它需要参考一些信息——这些信息就是**上下文**。

上下文包括：你刚刚问的问题、你之前对话里说过的话、你上传的文件、你设定的限制条件。

你可以把上下文想成一张**工作台**。

AI 要帮你干活，得先看见工作台上有什么。工作台上东西越清楚，它干活就越靠谱。

但这张工作台**有大小限制**。

如果对话太长、资料太多，最早的内容可能被"挤出"工作台——AI 看不到了，自然也想不起来了。

> 这就是为什么你跟 AI 聊着聊着，它突然忘了你之前说的要求。不是它变傻了，是工作台满了。

配图建议：上下文窗口 = 有限工作台示意图。素材：`assets/第03课_图2_上下文窗口像工作台.svg`

**给小白的三个实操建议：**

1. **一个任务开一个新对话**，不要什么都塞在同一个聊天里
2. **重要的要求放在最前面或最后面**，别埋在中间，AI 对开头和结尾的信息更敏感
3. **对话太长时，把核心要求重新发一遍**，相当于刷新它的"工作台"

## 6. 第四步：模型——AI 背后的"大脑"

上下文准备好了，Token 拆完了，接下来就轮到**模型**上场。

模型就是 AI 背后的"大脑"。

你打开 ChatGPT，是在和 OpenAI 的模型交互。打开 DeepSeek，是在和 DeepSeek 的模型交互。Kimi 是 Kimi 的模型，Claude 是 Anthropic 的模型。

不同模型，就像**不同能力的人**：
- 有的写作强
- 有的推理强
- 有的代码强
- 有的中文体验好
- 有的擅长处理长文档

所以模型不是越贵越好，也不是越新越适合你。

> 模型决定了 AI 的能力上限，但你的 Prompt 会影响它能发挥多少。

就像你请了一个很厉害的顾问。但你问题描述得很乱，顾问也很难直接给你想要的答案。

## 7. 第五步：API——让 AI 走出聊天窗口

最后一个词，API。

这个词对小白来说最容易显得吓人。但你可以先把它理解成一个**"接口"**。

平时你用 ChatGPT，是自己打开网页、手动输入问题。

API 做的事不同。它让**软件、网站和工作流自动去调用 AI**。

举几个你可能见过的例子：
- 一个写作工具里有"AI 改写"按钮——背后调用了 API
- 一个客服系统能自动回复用户——背后调用了 API
- 一个工作流能自动抓热点、总结内容、生成配图——背后也是 API

所以 API 的意义是：

> 让 AI 不只停留在聊天窗口里，而是进入真实的工具和流程。

小白不需要马上会写 API，但要知道它为什么重要。以后你用的越来越多 AI 功能，背后可能都是通过 API 调用的。

⚠️ **两个安全提醒：**

1. **API Key 不能随便给别人。** 它相当于你的调用权限。别人拿了可以用你的额度，甚至访问你的数据。
2. **通过 API 传的数据要注意隐私。** 客户资料、内部文件、个人敏感信息，传之前想想有没有风险。

## 8. 把五个词串成一条链路

现在我们把这五个词串起来：

你给 AI 发了一段 **Prompt**（任务说明）→ 文字被拆成 **Token**（AI 读取的小颗粒）→ AI 结合当前**上下文**（它能看到的所有信息）→ 交给**模型**（它的大脑）去推理和生成 → 如果不是手动操作而是软件自动调用，背后用的就是 **API**。

配图建议：五个概念串联成链路的流程图。素材：`assets/第03课_图1_AI常见名词地图.svg`

一句话记忆：

> Prompt 是你怎么说，Token 是 AI 怎么读，上下文是 AI 当前知道什么，模型是 AI 用什么能力去推，API 是软件怎么调用 AI。

记住这条链路，以后你再看到工具介绍里写"支持 128K 上下文""按 Token 计费""开放 API""优化 Prompt"——全都能看懂了。

## 9. 总结

这一期帮你建立了一张 AI 名词地图。

五个词不是孤立的技术概念，它们是你每次使用 AI 时**背后真正在发生的事**。

你不需要全部背下来。但下次看到这些词的时候，你不再陌生，也知道它们和你的使用体验有什么关系。

## 10. 课后小任务

**练习一：让 AI 帮你复习**

```text
请用小白能听懂的话，分别解释模型、Token、上下文、Prompt、API。
每个词给我：一句话解释 + 一个生活类比 + 它和我日常使用 AI 有什么关系。
```

**练习二：让 AI 串联起来**

```text
请把这五个词串成一条完整链路，解释我向 AI 提问时，背后大概发生了什么。
```

你不用一下子全记住。先让这些词和你的使用经验连起来。

下次看到有人说"Token 不够了""上下文太长了""换个模型试试"，你就知道他在说什么了。
