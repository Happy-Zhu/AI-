# 第 06 课：提示词不是咒语：让 AI 听懂你的结构化指令

## 1. 提示词不是玄学

很多人学 AI，一上来就找：

“万能提示词”“神级 Prompt”“一键爆款模板”。

这些模板不是完全没用。

但如果你只会复制，不理解背后的结构，很快就会失效。

> Prompt 的本质不是咒语，而是任务说明。

你把任务说明得越清楚，AI 越容易给你有用结果。

## 2. 为什么“帮我写一篇文章”效果差？

看这个提示词：

```text
帮我写一篇关于 AI 的文章。
```

它的问题不是短，而是不完整。

| 缺失信息 | AI 会怎么猜 |
| --- | --- |
| 读者是谁 | 可能写成科普，也可能写成行业分析 |
| 用途是什么 | 可能像公众号，也可能像论文 |
| 风格是什么 | 可能很官方、很空 |
| 字数多少 | 可能太短或太长 |
| 输出格式 | 可能直接写正文，不给提纲 |

你不给边界，AI 就会自己补。

它补对了，你觉得好；补错了，你觉得它废。

但问题往往出在任务没有说清楚。

## 3. 好提示词的五个部分

小白先记住这个结构：

```text
角色 + 任务 + 背景 + 要求 + 格式
```

| 部分 | 作用 | 示例 |
| --- | --- | --- |
| 角色 | 让 AI 知道用什么身份回答 | 你是一位 AI 小白课程老师 |
| 任务 | 说明要做什么 | 写一段课程开场白 |
| 背景 | 补充对象和场景 | 听众是完全小白 |
| 要求 | 给出风格和限制 | 真诚、有干货、不贩卖焦虑 |
| 格式 | 规定输出形式 | 输出 3 个版本，每个 150 字 |

组合起来就是：

```text
你是一位擅长给 AI 小白讲课的老师。
请帮我写一段课程开场白。
课程主题是“普通人为什么要学 AI”。
听众是完全小白。
要求理性、真诚、有干货，不要贩卖焦虑。
输出 3 个版本，每个版本控制在 150 字以内。
```

这比“帮我写开场白”稳定得多。

## 4. 案例：把烂提示词改好

原始提示词：

```text
帮我写一篇 AI 工具介绍。
```

问题：太空。

改成结构化提示词：

```text
你是一位擅长给小白讲 AI 工具的课程老师。
请帮我写一篇“AI 工具怎么选”的课程文档提纲。
读者是完全 AI 小白。
要求：
1. 先按工具类型分类；
2. 每一类要说明适合什么场景；
3. 至少给一个真实案例；
4. 不要堆工具名，不要写成广告。
输出格式：
一级标题 + 二级标题 + 每节要讲的重点。
```

这个提示词好在哪里？

- 明确了角色；
- 明确了读者；
- 明确了输出不是正文，而是提纲；
- 明确要求有分类、场景和案例；
- 明确不要广告腔。

## 5. 三个可以直接用的模板

### 5.1 解释概念模板

```text
请用完全小白能听懂的话解释【概念】。
要求：
1. 一句话解释；
2. 一个生活类比；
3. 一个具体例子；
4. 它和普通用户有什么关系；
5. 最容易误解的地方是什么。
```

### 5.2 整理资料模板

```text
请阅读下面内容，帮我整理成：
1. 5 个核心观点；
2. 每个观点的原文依据；
3. 3 个需要进一步核验的信息；
4. 一段适合小白看的总结。
不要添加原文没有的信息。
```

### 5.3 写作辅助模板

```text
我想写一篇【主题】文章，读者是【人群】。
请先不要写正文。
请输出：
1. 5 个选题角度；
2. 每个角度适合怎么展开；
3. 你最推荐的角度和理由；
4. 对应文章提纲。
要求语言真诚，不要空话。
```

## 6. 提示词不是越长越好

提示词不是越长越好。

关键不是字数，而是信息质量。

一个好提示词应该满足三点：

- 目标清楚；
- 边界清楚；
- 输出清楚。

无关信息不要乱塞。

比如你让 AI 写课程稿，重要的是听众、主题、风格、结构，不是你今天几点起床。

## 7. 本节小结

提示词不是玄学咒语，而是结构化任务说明。

先记住五要素：角色、任务、背景、要求、格式。

真正好的提示词，不是特别长，而是关键信息清楚。

## 8. 课后小任务

把这句：

```text
帮我写一篇文章。
```

改成一条完整提示词。

要求必须包含：角色、任务、背景、要求、格式。
